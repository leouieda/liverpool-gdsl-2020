Documentation is crucial

  • Undocumented code will not be widely used
  • Writing docs makes your code better
  • Forces to think clearly about design
  • ... and use your own code in examples
  • Document before you implement
  • "DDD: documentation driven development"

Code checks ()

  • For every commit/PR
  • Check code style (black)
  • Run tests (pytest)
  • Check test coverage (codecov)
  • Build the docs (sphinx)
  • Multiple combinations of OS, Python, dependencies

Deployment ()

  • New tag created on GitHub (v1.0.0)
  • Build HTML docs in new folder and push to gh-pages
  • Point /latest to the new folder
  • Build source/wheel and push to PyPI
  • conda-forge bot picks it up from there
